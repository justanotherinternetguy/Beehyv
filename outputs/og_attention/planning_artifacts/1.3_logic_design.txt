[Logic Analysis]
- ['dataset_loader.py', 'Implements DatasetLoader class to handle all data-related operations. Dependencies: torch, torch.utils.data, sacremoses, numpy, pyyaml, os, re. Classes/methods: - DatasetLoader.__init__(config: dict): initializes config, src/tgt vocab dicts, Moses tokenizer via sacremoses. - DatasetLoader.build_vocab(corpus: List[str], max_size: int = 30000): builds vocabulary with special tokens (<start>:0, <end>:1, [UNK]:2) plus top max_size-3 frequent words, saves vocab to file. - DatasetLoader.tokenize(text: str) -> List[str]: uses Moses tokenizer with no lowercasing/stemming per paper specs. - DatasetLoader.load_data() -> Tuple[DataLoader, DataLoader, DataLoader]: loads WMT 2014 parallel corpora, applies Axelrod et al. (2011) data selection to reduce to 348M words, tokenizes all text, filters training data into 30/50 word length variants (both source and target ≤ max length), splits into train (filtered), dev (news-test-2012+2013), test (news-test-2014). Returns PyTorch DataLoaders with custom collate_fn that pads sequences, and implements minibatch sorting logic (every 20 updates retrieve 1600 sentences, sort by max(source_len, target_len), split into 20 minibatches of 80). - DatasetLoader.collate_batch(batch: List[Tuple[List[int], List[int]]]) -> Tuple[Tensor, Tensor, Tensor, Tensor]: pads source/target sequences in batch, returns padded src, src_lengths, padded tgt, tgt_lengths.']
- ['model.py', "Defines all model components matching the paper's exact architecture. Dependencies: torch, torch.nn, numpy. No internal project dependencies. Classes/methods: - GatedRNNCell: implements Cho et al. 2014a gated hidden unit (pre-GRU). __init__(input_dim: int, hidden_dim: int, context_dim: Optional[int] = None): initializes weights per Appendix B.1 (recurrent weights orthogonal, others Gaussian). forward(x: Tensor, s_prev: Tensor, c: Optional[Tensor] = None) -> Tensor: computes reset gate r, update gate z, candidate state ã, new state s_new. - AlignmentModel: feedforward alignment model for RNNsearch. __init__(hidden_dim: int = 1000, context_dim: int = 2000): initializes W_a (1000x1000), U_a (1000x2000), v_a (1000). forward(s_prev: Tensor, h_j: Tensor) -> Tensor: computes e_ij = v_a^T tanh(W_a s_prev + U_a h_j). - MaxoutOutputLayer: output layer with maxout activation. __init__(hidden_dim: int = 1000, maxout_dim: int = 500, vocab_size: int = 30000): initializes U_o, V_o, C_o, W_o. forward(s_prev: Tensor, y_prev_emb: Tensor, c: Tensor) -> Tensor: computes combined feature t_i, applies maxout (split into 500 pairs, take max), softmax over vocab. - RNNEncDec: baseline RNN Encoder-Decoder. __init__(src_vocab_size: int = 30000, tgt_vocab_size: int = 30000, emb_dim: int = 620, hidden_dim: int = 1000, maxout_dim: int = 500): initializes src/tgt embeddings, forward RNN encoder, decoder (gated cell), output layer. encode(src: Tensor, src_lengths: Tensor) -> Tensor: encodes source to fixed context vector c = last encoder hidden state. decode(s_prev: Tensor, y_prev: Tensor, c: Tensor) -> Tuple[Tensor, Tensor]: decodes one step, returns logits and new state. forward(src: Tensor, tgt: Tensor, src_lengths: Tensor) -> Tensor: teacher forcing forward pass, returns logits for all target steps. - RNNsearch: proposed attention model. __init__(src_vocab_size: int = 30000, tgt_vocab_size: int = 30000, emb_dim: int = 620, hidden_dim: int = 1000, maxout_dim: int = 500): initializes src/tgt embeddings, forward/backward encoder RNNs, decoder, alignment model, output layer. encode(src: Tensor, src_lengths: Tensor) -> List[Tensor]: returns list of concatenated forward/backward annotations h_j (2000 dim). decode(s_prev: Tensor, y_prev: Tensor, annotations: List[Tensor]) -> Tuple[Tensor, Tensor, Tensor]: computes alignment weights α_ij, context c_i, decodes one step, returns logits, new state, α_ij. forward(src: Tensor, tgt: Tensor, src_lengths: Tensor) -> Tensor: teacher forcing forward pass with per-step attention, returns logits for all target steps."]
- ['trainer.py', 'Handles model training per paper specifications. Dependencies: torch, torch.nn, torch.optim, tqdm, numpy, model.py, dataset_loader.py. Classes/methods: - Trainer.__init__(model: object, train_loader: DataLoader, dev_loader: DataLoader, config: dict): initializes model, data loaders, Adadelta optimizer (eps=1e-6, rho=0.95), gradient clip norm 1.0, training config. - Trainer.train() -> None: runs training loop, iterates over epochs, every 20 updates retrieves 1600 sentences from train_loader, sorts by max length, splits into 20 minibatches of 80. For each batch: forward pass (teacher forcing), computes NLL loss, backpropagates, clips gradient L2 norm to 1.0, optimizer step. Validates on dev set every 1000 updates, saves best model by BLEU score. Trains RNNsearch-50 until validation BLEU stops improving (early stopping). - Trainer.validate() -> float: runs beam search decoding on dev set, computes BLEU score via sacrebleu.']
- ['evaluation.py', 'Handles model evaluation and qualitative analysis. Dependencies: torch, sacrebleu, matplotlib, numpy, model.py, dataset_loader.py. Classes/methods: - Evaluation.__init__(model: object, test_loader: DataLoader, config: dict): initializes model, test loader, beam width (default 10), no-UNK flag. - Evaluation.evaluate() -> dict: generates translations for all test sentences via beam search, computes BLEU score on full test set and filtered no-UNK subset (no UNK in source/reference, model blocked from generating UNK). Returns dict with bleu_all, bleu_no_unk. - Evaluation.beam_search(src: Tensor, src_lengths: Tensor) -> List[int]: implements beam search with given width, decodes until <end> is generated or max length reached, returns list of token IDs. - Evaluation.visualize_alignments(src_tokens: List[str], tgt_tokens: List[str], alpha_weights: Tensor) -> None: plots grayscale heatmap of α_ij weights, saves to file.']
- ['main.py', 'Entry point for the experiment. Dependencies: all project files, pyyaml, argparse. Classes/methods: - Main.__init__(config_path: str): loads config from yaml, parses command line args (model type, length variant). - Main.run_experiment() -> None: initializes DatasetLoader, loads data, initializes Model (RNNEncDec or RNNsearch), initializes Trainer, trains model, initializes Evaluation, evaluates model, saves results (BLEU scores, alignment plots). - __main__ block: parses args, creates Main instance, runs experiment.']

[Shared Knowledge]
1. Shared configuration variables loaded from config.yaml: vocab_size=30000, emb_dim=620, hidden_dim=1000, maxout_dim=500, minibatch_size=80, adadelta_eps=1e-6, adadelta_rho=0.95, clip_norm=1.0, beam_width=10, max_sentence_lengths=[30,50], special token IDs (<start>:0, <end>:1, [UNK]:2). 2. Weight initialization rules from Appendix B.1: Recurrent weights (U, U_z, U_r for all RNNs) initialized as random orthogonal matrices; W_a, U_a (alignment model) initialized as N(0, 0.001²); v_a (alignment model output) and all bias vectors initialized to 0; all other weights initialized as N(0, 0.01²). Implemented in model.py's __init__ methods. 3. Minibatch sorting logic: Every 20 training updates, retrieve next 1600 sentence pairs from shuffled training set, sort by max(source_length, target_length) descending, split into 20 minibatches of 80. Implemented in dataset_loader.py and used in trainer.py. 4. Special tokens are shared between dataset_loader.py (vocab building), model.py (embedding layers, decoder initialization), and evaluation.py (beam search decoding, UNK blocking).

[Anything UNCLEAR]
1. Beam search width is not specified in the paper; we assume default width 10, need confirmation. 2. Sentence length filtering for 30/50 word variants: paper does not specify if length refers to source, target, or both; we assume both source and target ≤ max length, need confirmation. 3. RNNencdec initial decoder state: paper does not specify, we follow Cho et al. 2014a: s₀ = tanh(W_s * c) where c is encoder's last hidden state, need confirmation. 4. Axelrod et al. (2011) data selection parameters to reduce 850M to 348M words are not provided; we use Fairseq's default implementation, need parameter clarification. 5. Special tokens (<start>, <end>, [UNK]) inclusion in 30k vocabulary: we assume they are added on top of 30k most frequent words, need confirmation. 6. RNNencdec context vector: paper states c = h_Tx (last forward hidden state), need confirmation no backward RNN is used for baseline. 7. Training hardware: paper uses Titan Black/Quadro K-6000 GPUs for ~5 days per model; we need to adjust epoch count for modern hardware. 8. WMT 2014 corpus file paths: need exact paths to each parallel corpus file to automate loading. 9. Moses tokenizer compatibility: paper uses original Perl Moses tokenizer; we use Python sacremoses port, need to confirm tokenization parity.
